# NCFE Web

Production frontend for **NCFE — National Centre for Financial Education**, the
financial-literacy assessment portal. Built with **Next.js 15 (App Router) +
React 19 + TypeScript**, with a security-first, SEO-first setup suitable for a
government platform.

## Getting started

```bash
npm install
cp .env.example .env.local   # set NEXT_PUBLIC_SITE_URL
npm run dev                  # http://localhost:3000
```

> ⚠️ Run only **one** `next dev` at a time across this repo — multiple dev
> servers corrupt the `.next` cache and serve unstyled pages.

Scripts: `dev`, `build`, `start`, `lint`, `typecheck`.

## Folder architecture

```
src/
  app/                 Routing layer only (thin). Pages compose components.
    layout.tsx         Root layout: font, <head> metadata, JSON-LD, chrome.
    page.tsx           Home page (Hero + TrustBanner + HomeSections).
    sitemap.ts         /sitemap.xml
    robots.ts          /robots.txt
    manifest.ts        /manifest.webmanifest (PWA)
    not-found.tsx      404
  middleware.ts        Per-request nonce-based Content-Security-Policy.
  components/
    layout/            Site chrome: Header, Footer (+ co-located CSS).
    home/              Home sections: Hero, TrustBanner, HomeSections.
  config/
    site.ts            Single source of truth: metadata, nav, SEO keywords.
  lib/
    seo.ts             buildMetadata() helper for consistent page metadata.
    structured-data.ts Schema.org JSON-LD (Organization, WebSite).
  styles/
    globals.css        Design tokens, reset, forms, buttons.
public/
  assets/              Images (logo, hero background).
  favicon.svg
```

Conventions:
- Import via the `@/*` alias (→ `src/*`).
- Components are **server components by default**; add `'use client'` only when
  state/effects/browser APIs are needed (e.g. `Header`).
- All page metadata flows through `buildMetadata()` so canonical URLs, Open
  Graph, Twitter cards and robots directives stay consistent.

## Security

- **Content-Security-Policy** — per-request nonce + `strict-dynamic`
  (`src/middleware.ts`). Inline scripts without the nonce cannot execute.
- **HTTP security headers** (`next.config.mjs`): HSTS (preload),
  `X-Frame-Options: DENY`, `X-Content-Type-Options: nosniff`,
  `Referrer-Policy`, a locked-down `Permissions-Policy`, and COOP.
- `poweredByHeader: false` — framework/version not advertised.
- Self-hosted fonts via `next/font` (no third-party CDN requests).
- No secrets in client code; only `NEXT_PUBLIC_*` env vars reach the browser.

## SEO

- Metadata API with title template, canonical URLs, Open Graph + Twitter cards.
- `sitemap.xml`, `robots.txt`, and a PWA manifest generated by the App Router.
- Schema.org JSON-LD for the organisation and website.
- Targeted financial-literacy keywords centralised in `config/site.ts`.
- Semantic HTML, a single `<h1>` per page, accessible landmarks, skip link,
  and `next/image` for optimised, responsive imagery.
